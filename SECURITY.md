# Security Policy

Thank you for helping keep ANIDA Digital Dojo and its visitors safe.

## Supported Version

This repository contains a static portfolio website. Only the latest version deployed from the `main` branch is currently supported with security updates.

| Version | Supported |
| ------- | --------- |
| `main`  | ✅ Yes    |
| Older commits or copies | ❌ No |

## Reporting a Vulnerability

Please do **not** report security vulnerabilities through public GitHub Issues.

Use GitHub's private vulnerability reporting feature:

1. Open the **Security** tab of this repository.
2. Select **Advisories**.
3. Click **Report a vulnerability**.

Please include:

- A clear description of the vulnerability.
- The affected page, URL, or file.
- Steps required to reproduce the issue.
- The possible security impact.
- Screenshots or proof of concept, if available.
- Suggested remediation, if known.

## Response Time

- Initial acknowledgement: within 72 hours.
- Preliminary assessment: within 7 days.
- Security updates will be provided when significant progress is made.

## Scope

The following are considered in scope:

- The website hosted at `https://builtbyanas.com/`.
- Source code contained in this repository.
- Exposed credentials, tokens, or sensitive information.
- Cross-site scripting, unsafe redirects, or unauthorized code execution.
- Security misconfigurations directly controlled by this project.

The following are out of scope:

- GitHub, GitHub Pages, or Cloudflare infrastructure.
- Denial-of-service or traffic flooding tests.
- Social engineering, phishing, or spam.
- Automated scanner reports without a demonstrated security impact.
- Attacks that affect other users, compromise privacy, or disrupt the website.

## Responsible Disclosure

Please allow reasonable time for the vulnerability to be investigated and fixed before publishing any details publicly.

Do not access, modify, download, or delete data that does not belong to you.

## Safe Harbor

Good-faith security research that follows this policy will be considered authorized. Researchers must avoid privacy violations, service disruption, data destruction, and exploitation beyond what is necessary to demonstrate the vulnerability.
